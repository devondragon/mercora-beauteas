<task type="auto">
  <name>Task 1: Create JSON-LD builder functions library</name>
  <files>lib/seo/json-ld.ts</files>
  <action>
Create `lib/seo/json-ld.ts` with the following exports. Follow project conventions (file header comment with === Module Name === style, JSDoc on each function, named exports).

1. **`JsonLdScript` component** -- a React server component that renders a JSON-LD script tag. Uses `JSON.stringify(data).replace(/&lt;/g, "\\u003c")` for XSS prevention per official Next.js docs -- this escapes HTML angle brackets so injected script tags in product names cannot execute. Add an eslint-disable comment for the dangerouslySetInnerHTML line noting this is the official Next.js pattern.

2. **`buildProductJsonLd(product, url)`** -- builds a schema.org Product object:
   - Import `resolveLocalizedField` and `resolveImageUrl` from `@/lib/seo/metadata`.
   - Extract `name` via `resolveLocalizedField(product.name)`.
   - Extract `description` via `resolveLocalizedField(product.description)`.
   - Resolve images: try `resolveImageUrl(product.primary_image)`, then map over `product.media` for additional images. Collect non-null results into `string[]`.
   - Find default variant: `product.variants?.find(v => v.id === product.default_variant_id) || product.variants?.[0]`.
   - Build `offers` from default variant:
     - **CRITICAL: Price is in cents.** Convert: `(variant.price.amount / 100).toFixed(2)`.
     - `priceCurrency`: `variant.price.currency || "USD"`.
     - Determine `availability`: if no inventory tracking or quantity > 0 use `"https://schema.org/InStock"`, if allow_backorder use `"https://schema.org/BackOrder"`, else `"https://schema.org/OutOfStock"`.
     - If multiple variants exist with differing prices, use `"@type": "AggregateOffer"` with `lowPrice`/`highPrice`. Otherwise use `"@type": "Offer"` with `price`.
   - Build `aggregateRating` only if `product.rating && product.rating.count > 0`.
   - Set `brand` to `{ "@type": "Brand", name: product.brand || "BeauTeas" }`.
   - Return `{ "@context": "https://schema.org", "@type": "Product", ... }`.

3. **`buildOrganizationJsonLd()`** -- builds a schema.org Organization object:
   - Import `brand` from `@/lib/brand`.
   - Import `BASE_URL` from `@/lib/seo/metadata`.
   - Collect social links: `Object.values(brand.social).filter(Boolean)`.
   - Return `{ "@context": "https://schema.org", "@type": "Organization", name, description, url, logo, sameAs }`.
   - Only include `sameAs` if socialLinks has entries.
   - Use `logo: BASE_URL + "/favicon.ico"`.

4. **`buildBreadcrumbJsonLd(items)`** -- builds a schema.org BreadcrumbList:
   - Accept `items: Array<{ name: string; url?: string }>`.
   - Map to `itemListElement` with `"@type": "ListItem"`, `position` (1-based), `name`, and `item` (URL, only if provided -- omit for last item per Google spec).
   - Return `{ "@context": "https://schema.org", "@type": "BreadcrumbList", itemListElement }`.

5. Define TypeScript interfaces for each JSON-LD structure inline. Do NOT use `schema-dts` package (project has no-new-dependencies constraint). Keep interfaces private to the module; only export the builder functions and JsonLdScript.
  </action>
