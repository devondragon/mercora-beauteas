  <name>Task 1: Create dynamic sitemap replacing static XML</name>
  <files>app/sitemap.ts, public/sitemap.xml</files>
  <action>
1. Create `app/sitemap.ts` as the default export async function returning `MetadataRoute.Sitemap`.

2. Import `listProducts` from `@/lib/models`, `listCategories` from `@/lib/models/mach/category`, and `getPublishedPages` from `@/lib/models/pages`.

3. Define `const BASE_URL = "https://beauteas.com"` at module level.

4. Inside the function, fetch all three data sources in parallel with `Promise.all`:
   - `listProducts({ status: ["active"] })`
   - `listCategories({ status: "active" })`
   - `getPublishedPages()`

5. Map products to sitemap entries:
   - URL: `${BASE_URL}/product/${typeof p.slug === "string" ? p.slug : p.id}`
   - lastModified: `p.updated_at ? new Date(p.updated_at) : new Date()`
   - changeFrequency: `"weekly"`
   - priority: `0.8`

6. Map categories to sitemap entries:
   - URL: `${BASE_URL}/category/${typeof c.slug === "string" ? c.slug : typeof c.slug === "object" ? Object.values(c.slug)[0] : c.id}`
   - lastModified: `c.updated_at ? new Date(c.updated_at) : new Date()`
   - changeFrequency: `"weekly"`
   - priority: `0.9`

7. Map CMS pages to sitemap entries:
   - URL: `${BASE_URL}/${page.slug}`
   - lastModified: `page.updated_at ? new Date(page.updated_at) : new Date()`
   - changeFrequency: `"monthly"`
   - priority: `0.6`

8. Add static entries: homepage (priority 1.0, weekly), with the homepage as the first item in the returned array.

9. Return `[homepage, ...categoryUrls, ...productUrls, ...pageUrls]`.

10. **CRITICAL:** Delete `public/sitemap.xml`. If both exist, the static file takes precedence and the dynamic route never runs. The `public/robots.txt` references `/sitemap.xml` which will still work since `app/sitemap.ts` serves at the same path.

11. Add a file header comment following project conventions (=== Module Name === style).

Do NOT use any external packages. Use only `import type { MetadataRoute } from "next"`.

<task type="auto">
  <name>Task 2: Add Shopify URL redirect rules to next.config.ts</name>
  <files>next.config.ts</files>
  <action>
1. Add an `async redirects()` method to the existing `nextConfig` object in `next.config.ts`, alongside the existing `headers()` method.

2. Return an array of three redirect rules:
   ```typescript
   async redirects() {
     return [
       {
         source: "/products/:slug",
         destination: "/product/:slug",
         permanent: true,
       },
       {
         source: "/collections/:slug",
         destination: "/category/:slug",
         permanent: true,
       },
       {
         source: "/pages/:slug",
         destination: "/:slug",
         permanent: true,
       },
     ];
   },
   ```

3. `permanent: true` produces a 308 redirect. Google treats 308 identically to 301 for SEO. Do NOT use `statusCode: 301` -- the Next.js `permanent` flag is the standard approach.

4. Do NOT modify any other part of `next.config.ts` (preserve images, experimental, webpack, headers config). Add `redirects` as a new property of `nextConfig`.

5. These are stub redirects for Phase 1. They handle the structural URL pattern differences between Shopify and Mercora (pluralized paths, /collections vs /category). Phase 4 (MIGR-08) will add any product-specific slug mapping if Shopify slugs differ from Mercora slugs.
