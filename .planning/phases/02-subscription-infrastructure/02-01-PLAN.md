2. Create `lib/db/schema/subscription.ts` with three Drizzle table definitions following the reviews.ts pattern:

   **subscription_plans table:**
   - `id`: text PK with `$defaultFn(() => \`PLN-${nanoid(8).toUpperCase()}\`)`
   - `product_id`: text NOT NULL
   - `frequency`: text NOT NULL with enum `['biweekly', 'monthly', 'bimonthly']`
   - `discount_percent`: integer NOT NULL default 10
   - `stripe_price_id`: text (nullable -- set when Stripe price is created)
   - `is_active`: integer boolean NOT NULL default true
   - `created_at`: text default CURRENT_TIMESTAMP
   - `updated_at`: text default CURRENT_TIMESTAMP
   - Indexes: `idx_sub_plans_product` on product_id

   **customer_subscriptions table:**
   - `id`: text PK with `$defaultFn(() => \`SUB-${nanoid(8).toUpperCase()}\`)`
   - `customer_id`: text NOT NULL (references customers table conceptually, but do NOT use .references() since the customer table uses dynamic IDs that may not exist yet during migration)
   - `plan_id`: text NOT NULL
   - `stripe_subscription_id`: text NOT NULL UNIQUE
   - `stripe_customer_id`: text NOT NULL
   - `status`: text NOT NULL default 'active' with enum `['active', 'paused', 'canceled', 'past_due', 'incomplete', 'trialing']`
   - `current_period_start`: text (nullable)
   - `current_period_end`: text (nullable)
   - `cancel_at_period_end`: integer boolean default false
   - `pause_collection`: text (nullable -- JSON of Stripe pause_collection object)
   - `created_at`: text default CURRENT_TIMESTAMP
   - `updated_at`: text default CURRENT_TIMESTAMP
   - `canceled_at`: text (nullable)
   - Indexes: `idx_cust_subs_customer` on customer_id, `idx_cust_subs_stripe_id` on stripe_subscription_id, `idx_cust_subs_status` on status

   **subscription_events table:**
   - `id`: text PK with `$defaultFn(() => \`EVT-${nanoid(8).toUpperCase()}\`)`
   - `subscription_id`: text NOT NULL (references customer_subscriptions.id)
   - `event_type`: text NOT NULL with enum `['created', 'renewed', 'payment_failed', 'paused', 'resumed', 'skipped', 'canceled', 'updated']`
   - `stripe_event_id`: text (nullable)
   - `details`: text (nullable -- JSON for event-specific data)
   - `created_at`: text default CURRENT_TIMESTAMP
   - Indexes: `idx_sub_events_subscription` on subscription_id, `idx_sub_events_type` on event_type

   Export row types using `$inferSelect` and `$inferInsert` patterns.

Implement all functions listed in the behavior section. Key implementation notes:
