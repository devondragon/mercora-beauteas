  <name>Task 1: Extend subscription model layer with admin query functions</name>
  <files>lib/models/mach/subscriptions.ts</files>
  <action>
Add the following new functions to the existing subscriptions model file. Preserve ALL existing functions and imports.

**1. `listSubscriptionsAdmin(options)`** - Paginated subscription list with joined data for the admin table.
- Parameters: `{ limit: number; offset: number; status?: string; search?: string }`
- Join customer_subscriptions with subscription_plans (on plan_id), products (on subscription_plans.product_id), and customers (on customer_id)
- Return items array with: subscription fields + plan frequency + plan discount_percent + product name (JSON localized) + product slug + customer person (JSON) + first product variant price (for subscription price calculation)
- For search: use SQL LIKE on the customers.person field (contains name/email as JSON) -- `customers.person LIKE '%searchterm%'`
- For status filter: apply eq condition on customer_subscriptions.status (skip if status is 'all' or undefined)
- Order by created_at DESC
- Also return total count for pagination meta
- Use raw SQL with `db.all()` or Drizzle `sql` template for the multi-table join since Drizzle's join builder doesn't handle 3+ table joins cleanly in D1

**2. `getAdminSubscriptionStats()`** - Extended KPI metrics for dashboard stat cards.
- Returns object with: activeCount, pausedCount, mrr, churnRate, newThisMonth, revenueTrendPercent
- Active/paused counts: use existing getSubscriptionStats() approach (COUNT with CASE WHEN)
- MRR calculation: For each active subscription, join to subscription_plans for discount_percent and products for variant price. MRR = SUM(variant_price_amount * (1 - discount_percent/100)). Since variant prices are stored as JSON Money objects `{"amount": 2499, "currency": "USD"}`, fetch active subs with plan+product+variant joins, then compute in JS. Convert cents to dollars for display (amount / 100).
- Churn rate (30-day rolling): Count subscription_events with event_type='canceled' in last 30 days. startActive = currentActive + canceledInPeriod. churnRate = startActive > 0 ? (canceledInPeriod / startActive) * 100 : 0
- New this month: Count customer_subscriptions where created_at >= first day of current month (use `new Date()` to calculate)
- Revenue trend: Compare current month MRR estimate to last month. Calculate as percentage change. Use subscription created_at dates to estimate. Simplify: count active subs created before this month vs total active subs.

**3. `getSubscriptionDetail(subscriptionId)`** - Full subscription detail with plan, product, and customer data for detail page.
- Single subscription by ID joined with subscription_plans, products, customers
- Return: subscription fields + plan fields + product name/slug + customer person data + first variant price
- Use same join strategy as listSubscriptionsAdmin but for single row

**4. `updateSubscriptionPlan(planId, updates)`** - Update existing plan's discount_percent or is_active status.
- Parameters: planId: string, updates: { discount_percent?: number; is_active?: boolean }
- Update subscription_plans table, set updated_at to new Date().toISOString()

**5. `getPlansWithSubscriberCount(productId)`** - List plans for a product with active subscriber count.
- For the product editor: return all plans (including inactive) for a productId
- Include a count of active customer_subscriptions per plan (LEFT JOIN + COUNT)
- Return: plan fields + activeSubscriberCount

Add these new imports at the top: `count` from drizzle-orm, and import `customers` from `@/lib/db/schema/customer`, `products` and `product_variants` from `@/lib/db/schema/products`.

IMPORTANT pitfalls to handle:
- Product names are JSON localized: `{"en": "Green Tea"}` -- parse with JSON.parse(name).en in application code, not in SQL
- Variant prices are JSON Money: `{"amount": 2499, "currency": "USD"}` -- parse in JS after query
- Customer person field is JSON -- parse in JS for name/email extraction
- Division by zero guard on churn rate
- Null guards on all JSON parse operations (try/catch or optional chaining)

**A. Create `app/api/admin/subscriptions/route.ts`** - GET endpoint for subscription list + stats.

Follow the exact pattern from `app/api/admin/reviews/route.ts`:
1. Import `checkAdminPermissions` from `@/lib/auth/admin-middleware`
2. Export async function GET(request: NextRequest)
3. Auth check first: `const auth = await checkAdminPermissions(request); if (!auth.success) return 401`
4. Parse searchParams: limit (default 20, max 100), offset (default 0), status (optional), search (optional)
5. Call `listSubscriptionsAdmin({ limit, offset, status, search })` for paginated data
6. Call `getAdminSubscriptionStats()` for dashboard KPIs
7. Return JSON: `{ success: true, data: items, stats: { activeCount, pausedCount, mrr, churnRate, newThisMonth, revenueTrendPercent }, meta: { total, limit, offset } }`
8. Wrap in try/catch, return 500 on error

**B. Create `app/api/admin/subscriptions/[id]/route.ts`** - GET endpoint for single subscription detail.

1. Auth check
2. Extract id from params (Next.js App Router dynamic route: `{ params }: { params: Promise<{ id: string }> }`, await params)
3. Call `getSubscriptionDetail(id)` for subscription + plan + product + customer
4. Call `getSubscriptionEvents(id)` for event timeline
5. Return JSON: `{ success: true, data: { subscription, events } }`
6. If subscription not found, return 404: `{ success: false, error: "Subscription not found" }`

**C. Create `app/api/admin/subscriptions/plans/route.ts`** - GET and POST endpoints for plan management.

GET:
1. Auth check
2. Parse productId from searchParams (required)
3. Call `getPlansWithSubscriberCount(productId)`
4. Compute hasActiveSubscribers: any plan with activeSubscriberCount > 0
5. Return: `{ success: true, plans, hasActiveSubscribers }`

POST:
1. Auth check
2. Parse JSON body: `{ productId: string, plans: Array<{ frequency, discount_percent, is_active }> }`
3. For each plan in the array:
   - If a plan for this product+frequency already exists (check via getPlansWithSubscriberCount), call `updateSubscriptionPlan()` to update discount_percent and is_active
   - If no plan exists for this frequency, call `createSubscriptionPlan()` to create it
4. Return: `{ success: true, message: "Plans updated" }`
5. Validation: discount_percent must be 0-100, frequency must be one of biweekly/monthly/bimonthly
