       - Status mapping: Shopify financial_status + fulfillment_status -> Mercora status:

         - fulfillment_status=partial -> 'shipped'
         - financial_status=paid + no fulfillment -> 'processing'
         - financial_status=refunded -> 'refunded'
         - financial_status=voided/cancelled OR cancelled_at set -> 'cancelled'
         - Otherwise -> 'pending'
       - total_amount: { amount: Math.round(parseFloat(total_price) * 100), currency: currency || 'USD' }
       - items: transform line_items to JSON array: [{ product_id: idMap.resolve('products', line_item.product_id), variant_id: idMap.resolve('variants', line_item.variant_id), name: line_item.title, quantity: line_item.quantity, price: { amount: Math.round(parseFloat(line_item.price) * 100), currency: 'USD' }, sku: line_item.sku }]
       - shipping_address/billing_address: transform to Mercora address JSON
       - payment_status: financial_status=paid -> 'paid', refunded -> 'refunded', pending -> 'pending', else 'failed'
       - external_references: { shopify_id: shopifyId, shopify_order_name: shopify.name (e.g., '#1001') }
       - extensions: { shopify_financial_status, shopify_fulfillment_status, shopify_tags }
       - Preserve original created_at/updated_at timestamps
       - Register: idMap.register('orders', shopifyId, mercoraOrderId)

       - Review ID: REV-{nanoid(10).toUpperCase()} (matching existing convention)
       - product_id: resolve via idMap.resolve('products', review.product_handle) -- use product_handle to find the Mercora product ID. If product_handle matches a Shopify handle that was used as the descriptive ID (e.g., cat_{handle}), try idMap lookup. Skip review if product not found.
       - order_id: 'IMPORTED' (synthetic value -- per research recommendation for NOT NULL constraint)
       - customer_id: try to match reviewer_email against idMap customer entries. If match found, use mapped Clerk user ID. Otherwise use 'IMPORTED'.

       - body: review.body

       - is_verified: true (they were published on Judge.me)
       - admin_response: review.reply (if present)
       - submitted_at: review.review_date (parse to ISO 8601)
       - published_at: review.review_date
       - metadata: JSON with { source: 'judgeme', original_product_handle: review.product_handle, picture_urls: review.picture_urls }
       - Register: idMap.register('reviews', `${review.product_handle}_${review.reviewer_email}_${review.review_date}`, reviewId)

    3. Create rating recalculation logic in migrate-reviews.ts (NOT a separate transformer -- runs after loading):
       - After all reviews loaded to D1, for each unique product_id in migrated reviews:
         - Query review counts and ratings from D1: SELECT rating, COUNT(*) as count FROM product_reviews WHERE product_id = ? AND status = 'published' GROUP BY rating
         - Calculate: average = sum(rating * count) / total_count, distribution = { 1: n, 2: n, 3: n, 4: n, 5: n }
         - Build Rating JSON: { average, count: total_count, distribution }
         - UPDATE products SET rating = '{json}' WHERE id = '{product_id}'
       - Use executeQuery() for SELECT queries and executeSql() for UPDATE

    4. Create `scripts/shopify-migration/migrate-reviews.ts`:
       - Standalone: npx tsx scripts/shopify-migration/migrate-reviews.ts

       - updated_at: Math.floor(new Date(shopify.updated_at).getTime() / 1000) (Unix timestamp)
       - version: 1
