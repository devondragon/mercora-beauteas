    1. Update `middleware.ts` to add D1 redirect lookup for Shopify URLs:
       - Import getDb (or getDbAsync) from '@/lib/db' and redirect_map from '@/lib/db/schema/redirect-map'
       - Import { eq } from 'drizzle-orm'
       - AFTER the maintenance mode try/catch block and BEFORE the final `return NextResponse.next()`:
       - Add a new try/catch block for redirect lookup:
         ```typescript
         try {
           // Check for Shopify URL redirects from migration data
           // Only check paths that could be old Shopify URLs (starts with /products/, /collections/, /pages/, or other known patterns)
           if (pathname.startsWith('/products/') || pathname.startsWith('/collections/') || pathname.startsWith('/pages/')) {
             const db = getDb();
             const [redirectRow] = await db
               .select({ target_path: redirect_map.target_path, status_code: redirect_map.status_code })
               .from(redirect_map)
               .where(eq(redirect_map.source_path, pathname))
               .limit(1);

             if (redirectRow) {
               return NextResponse.redirect(
                 new URL(redirectRow.target_path, req.url),
                 redirectRow.status_code
               );
             }
           }
         } catch (error) {
           // If redirect lookup fails, continue normally (next.config.ts structural redirects still work)
           console.error('Error checking redirect map:', error);
         }
         ```
       - NOTE: This runs AFTER next.config.ts redirects in the request pipeline. Next.js processes config redirects before middleware. So if a structural redirect like /products/:slug -> /product/:slug already handles the URL, middleware won't even see it. The D1 lookup catches URLs where slugs changed or where structural patterns don't apply.
       - IMPORTANT: Wrap in try/catch so redirect lookup failures never break the site (same pattern as maintenance mode check).
       - Do NOT modify the existing Clerk middleware setup, maintenance mode logic, or matcher config.
