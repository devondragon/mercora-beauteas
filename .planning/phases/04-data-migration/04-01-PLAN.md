       - Generates INSERT SQL via wrangler-exec helpers, chunked at 50 records
       - Clears existing demo data first: DELETE FROM {table} WHERE id LIKE '{prefix}%' for known prefixes (prod_, variant_, cat_)
       - Returns LoadResult with counts

    2. Create `scripts/shopify-migration/loaders/r2-loader.ts`:
       - uploadImages(images: Array<{ sourceUrl: string, r2Key: string }>, config): Promise<{ uploaded: number, failed: string[] }>
       - For each image: downloadWithRetry(sourceUrl) -> uploadToR2(r2Key, buffer, contentType)
       - Log failures but continue (per user decision: retry 3x with backoff, skip and log on failure)
       - Return counts of uploaded and list of failed URLs

    3. Create `scripts/shopify-migration/extractors/file-based/collections.ts`:
       - extractCollectionsFromFile(dataDir: string): Promise<ExtractResult<ShopifyCollection>>
       - Read from {dataDir}/collections.json or {dataDir}/collections.csv
       - JSON preferred; fall back to CSV if JSON not found

    4. Create `scripts/shopify-migration/extractors/shopify-api/collections.ts`:
       - extractCollectionsFromApi(client: ShopifyClient): Promise<ExtractResult<ShopifyCollection>>
       - Fetch both custom_collections and smart_collections, merge into unified ShopifyCollection array

    5. Create `scripts/shopify-migration/transformers/categories.ts`:
       - transformCategories(collections: ShopifyCollection[], idMap: IdMap): TransformResult<ShopifyCollection, MercoraCategoryInsert>
       - Generate descriptive IDs: cat_{slugified_handle} (matching existing convention)
       - Register in idMap: idMap.register('categories', shopifyId, mercoraId)
       - CRITICAL: Wrap name/description/slug in localization format: JSON.stringify({"en": value})
       - Map fields: handle -> slug (localized), title -> name (localized), body_html -> description (localized), image -> primaryImage (JSON Media object), published_at -> createdAt
       - Store shopify_id in externalReferences JSON
       - Set status to 'active' for published collections

    6. Create `scripts/shopify-migration/extractors/file-based/products.ts`:
       - extractProductsFromFile(dataDir: string): Promise<ExtractResult<ShopifyProduct>>
       - Read from {dataDir}/products.csv (Shopify export format)
       - Group rows by Handle (Shopify CSV has one row per variant, multiple rows per product)
       - Parse variant rows into product.variants array
       - Parse image rows into product.images array
       - Handle multi-row products where first row has product-level data and subsequent rows have only variant/image data

    7. Create `scripts/shopify-migration/extractors/shopify-api/products.ts`:
       - extractProductsFromApi(client: ShopifyClient): Promise<ExtractResult<ShopifyProduct>>
       - Fetch products with variants and images included

    8. Create `scripts/shopify-migration/transformers/products.ts`:
       - transformProducts(products: ShopifyProduct[], idMap: IdMap): TransformResult<ShopifyProduct, { product: MercoraProductInsert, variants: MercoraVariantInsert[], images: Array<{ sourceUrl: string, r2Key: string }> }>
       - Generate descriptive product IDs: prod_{slugified_handle} (matching existing convention like prod_morning)
       - Generate descriptive variant IDs: variant_{product_slug}_{sku_suffix} (matching existing convention like variant_morning_1box)
       - Register in idMap: idMap.register('products', shopifyId, mercoraId)
       - Price transformation: Shopify decimal string "14.99" -> Mercora Money { amount: 1499, currency: "USD" } (multiply by 100, round)
       - compare_at_price: same transformation for Variant Compare At Price
       - Options: Shopify Option1 Name/Value, Option2, Option3 -> Mercora options array with { id, name, type: 'select', values: [{id, value}] }
       - Variant option_values: map to [{option_id, value}] matching the product options
       - Images: generate R2 keys as products/{product-slug}.jpg, products/{product-slug}-2.jpg, etc. First image becomes primary_image as Media object
       - media: array of Media objects for all images
       - SEO: map Shopify SEO Title/Description to { title, description } JSON
       - Tags: split comma-separated Shopify tags into JSON array
       - Categories: resolve Shopify collection handles to Mercora category IDs via idMap.resolve('categories', handle)
       - status: Shopify 'active' -> 'active', 'draft' -> 'draft', 'archived' -> 'archived'
       - external_references: { shopify_id: shopifyId }
       - extensions: preserve useful Shopify data (vendor, type) in extensions JSON
       - Inventory: { track_inventory: true, quantity: parseInt(Variant Inventory Qty), allow_backorder: policy === 'continue' }
       - Weight: { value: parseInt(Variant Grams), unit: 'g' }
       - Set default_variant_id to first variant ID
       - All JSON fields must be JSON.stringify()'d for SQL insertion (products table uses text columns, not JSON mode for most fields)

        - Extract products -> Transform to products/variants/images -> Load products+variants to D1 -> Upload images to R2
        - Save updated idMap
        - Log results via logger including image upload counts
