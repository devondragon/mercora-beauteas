The component has two phases:

**Phase 1 -- Collect info and create SetupIntent:**
- Render form with: email (pre-filled from Clerk user), name (pre-filled from Clerk user), shipping address fields (street, city, state, zip, country -- same fields as existing ShippingForm)
- On submit, POST to /api/setup-intent with { email, name }
- On success, store clientSecret and customerId in state, advance to Phase 2

**Phase 2 -- Collect payment method:**
- Wrap content in StripeProvider with the clientSecret
- Create a nested PaymentFormInner component (must be inside StripeProvider to access useStripe/useElements)
- Render PaymentElement (same options as existing PaymentForm: layout tabs, payment method order card/apple_pay/google_pay)
- Show order summary sidebar: product name, product image, frequency label, subscription price, discount percentage
- Submit button: "Start Subscription"
- On submit, call stripe.confirmSetup({ elements, confirmParams: { return_url: window.location.origin + '/subscribe/confirmation' }, redirect: 'if_required' })
- CRITICAL: Use stripe.confirmSetup(), NOT stripe.confirmPayment() -- SetupIntents require confirmSetup
- If confirmSetup succeeds (no error):
  - POST to /api/subscriptions with { setupIntentId: setupIntent.id, planId: plan.id, shippingAddress: addressData }
  - On success, router.push('/subscribe/confirmation?plan=' + plan.id)
- If confirmSetup fails, show error toast
- Handle the case where Stripe redirects (e.g., 3D Secure) by checking the setup_intent query param on the confirmation page

Use dark theme consistent with the rest of the site: bg-neutral-900, text-white, neutral-800 borders, orange-500 accent.

Frequency label mapping: biweekly -> "Every 2 weeks", monthly -> "Monthly", bimonthly -> "Every 2 months"

Loading states: disable submit button and show spinner during API calls. Error states: show error message with red styling.
