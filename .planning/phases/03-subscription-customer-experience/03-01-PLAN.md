Create 7 API route files following the established API route pattern (try/catch, auth() from @clerk/nextjs/server, NextResponse.json).

**POST /api/setup-intent/route.ts:**
- Import auth from @clerk/nextjs/server, getStripeForWorkers from @/lib/stripe
- Require Clerk auth (return 401 if no userId)
- Accept JSON body: { email: string, name: string }
- Check if a Stripe Customer already exists for this userId by calling stripe.customers.list({ limit: 1, metadata: { clerk_user_id: userId } })
- If no customer found, create one: stripe.customers.create({ email, name, metadata: { clerk_user_id: userId } })
- Create SetupIntent: stripe.setupIntents.create({ customer: stripeCustomerId, usage: 'off_session', automatic_payment_methods: { enabled: true } })
- Return { clientSecret: setupIntent.client_secret, customerId: stripeCustomerId }

**GET /api/subscriptions/route.ts:**
- Require Clerk auth
- Call getSubscriptionsByCustomer(userId)
- For each subscription, also fetch the plan via getSubscriptionPlanById(sub.plan_id) to include plan details (frequency, discount)
- Return { subscriptions: enrichedSubscriptions }

**POST /api/subscriptions/route.ts:**
- Require Clerk auth
- Accept JSON body: { setupIntentId: string, planId: string, shippingAddress: object }
- Fetch the plan via getSubscriptionPlanById(planId) -- return 404 if not found
- Retrieve the SetupIntent from Stripe to get payment_method: stripe.setupIntents.retrieve(setupIntentId)
- Extract payment_method from setupIntent.payment_method (it's a string after confirmation)
- Get the Stripe customer ID from the SetupIntent: setupIntent.customer
- Create Stripe Subscription: stripe.subscriptions.create({ customer: stripeCustomerId, items: [{ price: plan.stripe_price_id }], default_payment_method: paymentMethodId, metadata: { customer_id: userId, plan_id: planId, product_id: plan.product_id } })
- The webhook handler (already built in Phase 2) will create the D1 record and send the email
- Return 201 with { subscription: { id: stripeSubscription.id, status: stripeSubscription.status } }

**POST /api/subscriptions/[id]/pause/route.ts:**
- Require Clerk auth. Extract { id } from params (params is Promise<{ id: string }>, await it).
- Ownership check: call getSubscriptionsByCustomer(userId), find sub where sub.id === id. Return 404 if not found.
- Call stripe.subscriptions.update(sub.stripe_subscription_id, { pause_collection: { behavior: 'void' } })
- Return { success: true }

**POST /api/subscriptions/[id]/resume/route.ts:**
- Same ownership check pattern
- Call stripe.subscriptions.update(sub.stripe_subscription_id, { pause_collection: '' }) -- empty string clears pause
- Return { success: true }

**POST /api/subscriptions/[id]/skip/route.ts:**
- Same ownership check pattern
- Retrieve subscription from Stripe: stripe.subscriptions.retrieve(sub.stripe_subscription_id)
- Get current_period_end from subscription.items.data[0]?.current_period_end (Stripe API 2025-08-27.basil puts period dates on SubscriptionItem, not Subscription root)
- Call stripe.subscriptions.update(sub.stripe_subscription_id, { pause_collection: { behavior: 'void', resumes_at: currentPeriodEnd } })
- Return { success: true }

**POST /api/subscriptions/[id]/cancel/route.ts:**
- Same ownership check pattern
- Call stripe.subscriptions.update(sub.stripe_subscription_id, { cancel_at_period_end: true })
- Return { success: true }

IMPORTANT: All routes go through Stripe API for state changes. The webhook handler (Phase 2) updates D1 and sends emails automatically. Do NOT update D1 directly from these routes.

IMPORTANT: For `params` in Next.js 15 dynamic routes, params is a Promise. Use: `const { id } = await params;`
