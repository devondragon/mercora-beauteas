### Integration Issues

**INT-01: Email manage URL broken** (functional)
- **From:** Phase 2 webhook handlers -> **To:** Phase 3 subscription management page
- **Problem:** All 6 webhook handlers hardcode `manageUrl: https://beauteas.com/account/subscriptions/${d1Sub.id}` but no route exists at `/account/subscriptions`. Actual page is `/subscriptions` (no detail view).
- **Impact:** Every subscription lifecycle email contains a broken "Manage Subscription" link (404)
- **Fix:** Change `manageUrl` to `https://beauteas.com/subscriptions` in subscription-handlers.ts and invoice-handlers.ts

**INT-02: Email product name shows internal IDs** (cosmetic)
- **From:** Phase 2 webhook handlers -> **To:** sendSubscriptionEmail()
- **Problem:** `handleSubscriptionCreated` passes `plan.product_id` as productName. All other handlers pass `d1Sub.plan_id`. Both are internal IDs, not human-readable names.
- **Impact:** Emails display "prod_jasmine-green" or "PLN-abc12345" instead of "Jasmine Green Tea"
- **Fix:** Add product name resolution by fetching product via `plan.product_id`
